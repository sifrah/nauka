The `service` module manages the WireGuard systemd service that keeps the mesh tunnel running persistently. It generates the WireGuard configuration file and a custom systemd unit, and provides operations to install, start, stop, and remove them.

:::note
This module is now a re-export of `fabric::wireguard::service`, which contains the canonical implementation. The two modules are identical — use whichever import path is convenient.
:::

## Service name and files

| Item             | Path                                     |
|------------------|------------------------------------------|
| Service name     | `nauka-wg`                               |
| WireGuard config | `/etc/wireguard/nauka0.conf`             |
| systemd unit     | `/etc/systemd/system/nauka-wg.service`   |

Nauka writes both the WireGuard config and a custom systemd unit (Type=oneshot, RemainAfterExit=yes) that wraps `wg-quick up/down`.

## The WireGuard config

`nauka0.conf` is generated from `FabricState`. A typical config looks like:

```ini
[Interface]
PrivateKey = <this node's WG private key>
ListenPort = 51820
Address = fd12:3456:789a::1/128

[Peer]
PublicKey = <peer pubkey>
AllowedIPs = fd12:3456:789a::2/128
PersistentKeepalive = 25
Endpoint = 203.0.113.20:51820
```

Every known peer gets a `[Peer]` section. `AllowedIPs` is set to the peer's single mesh IPv6 address (`/128`), which means only mesh traffic is routed over the tunnel — not the peer's entire internet traffic.

## Operations

### install

Writes `nauka0.conf` to `/etc/wireguard/` with `0600` permissions (readable only by root) and the systemd unit to `/etc/systemd/system/`. Called at the end of `init` and `join`, after the identity and peer list are ready.

### enable_and_start

Enables the service at boot and starts it immediately:

```sh
systemctl enable --now nauka-wg
```

### start / stop

Start or stop the service without changing the boot configuration.

### update_config

Rewrites `nauka0.conf` from the current `FabricState` and applies changes via `wg syncconf` (live reload without tearing down the interface). If `wg syncconf` fails, falls back to a full service restart.

### uninstall

Stops and disables the service, then removes both the unit file and `nauka0.conf`. Called by `nauka hv leave` as part of the full cleanup sequence.

## Permissions

All operations that write to `/etc/wireguard/` or invoke `systemctl` require root. Nauka expects to be run as root on hypervisor nodes.
