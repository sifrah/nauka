The `service` module handles the ZeroFS binary lifecycle and the per-region systemd services. Each configured region gets its own isolated ZeroFS instance managed by a dedicated service unit.

## Installation

`ensure_installed` checks for the ZeroFS binary at `/usr/local/bin/zerofs`. If it is missing, it attempts to download a pre-compiled release from GitHub:

```
https://github.com/barre/zerofs/releases/latest/download/zerofs-<arch>-unknown-linux-gnu
```

If the direct download fails, it falls back to `cargo install zerofs`. Both attempts must succeed before continuing. The binary is marked executable after download.

ZeroFS installation is triggered automatically when a storage region is first added to a node.

## Per-region configuration

`generate_config` produces a TOML file at `/etc/nauka/zerofs/<region>.toml` for each region. The config covers:

```toml
[storage]
type = "s3"
endpoint = "<s3_endpoint>"
bucket = "<s3_bucket>"
access_key = "<s3_access_key>"
secret_key = "<s3_secret_key>"

[cache]
path = "/var/lib/nauka/zerofs/<region>"
size = "10GB"

[encryption]
enabled = true

[compression]
algorithm = "lz4"

[nbd]
enabled = true

[server]
bind = "127.0.0.1"
```

S3 credentials are written directly into the TOML config file. The file is created with `0600` permissions (owner-readable only) to protect the credentials.

## Systemd units

Each region gets a systemd unit at `/etc/systemd/system/nauka-zerofs-<region>.service`. The unit:

- Depends on `network-online.target` and `nauka-wg.service` (the mesh must be up)
- Runs `zerofs run -c /etc/nauka/zerofs/<region>.toml`
- Restarts on failure with a 5-second backoff

## Service lifecycle

| Function | Effect |
|---|---|
| `install_region(config)` | Write TOML config + systemd unit, reload systemd daemon |
| `start_region(region)` | `systemctl enable --now nauka-zerofs-<region>` |
| `stop_region(region)` | `systemctl stop nauka-zerofs-<region>` |
| `is_region_active(region)` | Check if the service is currently running |
| `uninstall_region(region)` | Stop + disable, remove unit and config files, delete cache dir |
| `uninstall_all` | Discover and remove all `nauka-zerofs-*` units and their data |

Service names follow the pattern `nauka-zerofs-<region>`, e.g., `nauka-zerofs-eu` or `nauka-zerofs-us-east`.

## Directories

| Path | Purpose |
|---|---|
| `/etc/nauka/zerofs/` | Per-region TOML config files |
| `/var/lib/nauka/zerofs/<region>/` | Local read cache |
| `/mnt/nauka/<region>/` | Mount point (created but used by ZeroFS internally) |
