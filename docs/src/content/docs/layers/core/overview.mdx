The Core layer is the foundation of Nauka. It contains no I/O, no async code, and no external dependencies beyond well-audited libraries. Everything in Core is a pure type, a pure function, or a validated data structure.

Every other layer — State, Forge, Networking, Compute — imports from Core. Nothing in Core imports from any other Nauka layer. This makes Core the stable base that the rest of the system depends on.

## Why a dedicated Core layer?

Nauka is a distributed system. Errors, IDs, addresses, and validation logic appear everywhere. Without a shared foundation, these things end up duplicated, inconsistent, or subtly incompatible.

Core solves this by providing a single source of truth for:

- **How errors are represented** — every error has a code, a message, a suggestion, and optional context
- **How identities are structured** — every resource has a typed ULID that encodes what it is and when it was created
- **How addresses are assigned** — IPv6 ULA addressing derived deterministically from cryptographic keys
- **How configuration is loaded** — a consistent priority chain from file to env to CLI flags
- **How input is validated** — names, ports, regions, and durations validated at entry points before any state changes

## Design principles

**No I/O in Core.** Functions in Core never read files, open sockets, or spawn threads. This makes them trivially testable and composable. When I/O is needed, it happens in the layers above.

**Errors are actionable.** Every `NaukaError` carries a suggestion — a human-readable hint about what to do next. Errors are never just codes or raw strings.

**Types encode intent.** An `OrgId` and a `VpcId` are different types even though both are ULIDs. You cannot accidentally pass one where the other is expected. The type system enforces correctness at compile time.

**Validation at the boundary.** All user-supplied input is validated as soon as it enters the system — in the CLI, in the API, before any state changes are made. Validation rules live in Core so they are consistent everywhere.
