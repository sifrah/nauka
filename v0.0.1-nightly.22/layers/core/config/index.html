Configuration

Nauka’s configuration follows a clear priority chain. You can set values in a config file, override them with environment variables, and override those with CLI flags. The highest-priority source always wins.

Priority chain

Section titled “Priority chain”
flowchart TD
  CLI["CLI flags<br/>(highest priority)"] --> MERGED["Resolved<br/>configuration"]
  ENV["Environment variables<br/>(NAUKA_*)"] --> MERGED
  FILE["~/.nauka/config.toml"] --> MERGED
  DEFAULTS["Built-in defaults<br/>(lowest priority)"] --> MERGED

A value is only taken from a lower-priority source if no higher-priority source sets it. An absent CLI flag does not override a config file value.

Config file

Section titled “Config file”

The config file lives at ~/.nauka/config.toml. If the file does not exist, Nauka uses built-in defaults for all values. A complete default configuration looks like this:

The config file is saved with mode 0600 (owner read/write only) to protect any sensitive values.

Environment variables

Section titled “Environment variables”

Every configurable setting can be overridden with an environment variable using the NAUKA_ prefix:

Environment variableConfig path
NAUKA_LOG_LEVELlogging.level
NAUKA_LOG_FORMATlogging.format
NAUKA_LOG_FILElogging.file
NAUKA_WG_INTERFACEwireguard.interface_name
NAUKA_WG_PORTwireguard.listen_port
NAUKA_HEALTH_INTERVALdaemon.health_check_interval
NAUKA_RECONCILE_INTERVALdaemon.reconcile_interval
NAUKA_UNREACHABLE_TIMEOUTdaemon.unreachable_timeout
NAUKA_CACHE_MEMORY_MBstorage.cache_memory_mb
NAUKA_CACHE_DISK_GBstorage.cache_disk_gb

This makes Nauka easy to configure in containerized or automated environments without writing config files.

Configuration sections

Section titled “Configuration sections”

daemon

Section titled “daemon”

Controls health checking and reconciliation behavior.

KeyDefaultDescription
health_check_interval60sHow often to check peer health
reconcile_interval30sHow often to reconcile desired vs. actual state
persist_interval30sHow often to persist state to disk
unreachable_timeout5mTime before a peer is marked unreachable
max_concurrent_requests100Maximum concurrent API requests

wireguard

Section titled “wireguard”

Controls the WireGuard interface.

KeyDefaultDescription
interface_namenauka0Name of the WireGuard network interface
keepalive_interval25sWireGuard persistent keepalive interval
listen_port51820UDP port for WireGuard traffic

peering

Section titled “peering”

Controls the mesh join process.

KeyDefaultDescription
join_timeout5mMaximum time to wait for a join operation to complete
exchange_timeout30sTimeout for the key exchange phase
max_concurrent_connections100Maximum simultaneous incoming peering connections
max_pending_joins100Maximum join requests waiting for approval

storage

Section titled “storage”

Controls cache sizing.

KeyDefaultDescription
cache_memory_mb4096In-memory cache limit in megabytes
cache_disk_gb100On-disk cache limit in gigabytes

Setting both to 0 disables caching entirely, which Nauka will warn about.

logging

Section titled “logging”

Controls log output. See also Structured Logging for details on formats, rotation, and filtering.

KeyDefaultDescription
levelinfoLog level: trace, debug, info, warn, error
formattextOutput format: text (human-readable) or json (structured)
file(empty)Log file path. Empty means stderr only.
max_file_size_mb50Maximum log file size before rotation
max_files3Number of rotated log files to keep
