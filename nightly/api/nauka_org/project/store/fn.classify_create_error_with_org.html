Wrap crate::sdk_bridge::classify_create_error so the user-facing duplicate message includes the owning org. The composite unique index on (org, name) rejects (org_a, "web") + (org_a, "web") but allows (org_a, "web") + (org_b, "web"), so the error wording “project ‘web’ already exists in org ‘acme’” is both accurate and matches the pre-P2.10 wording the CLI tests expect.
