nauka_hypervisor::controlplane

Module ops 

Source
Expand description

Control plane operations — high-level orchestration.

Called by the hypervisor handler during init/join/leave/status. Orchestrates TiUP install → config → systemd → health check.

PD (Raft consensus) member count is configurable (1, 3, 5, 7) via --max-pd-members on init (default: 3). Nodes beyond the PD limit run TiKV only (storage), connecting to existing PD.
