async fn open_datastore_with_retry(path_str: &str) -> Result<Surreal<Db>>
Expand description

Open the SurrealKV datastore at path_str, retrying on flock contention until a 5-second deadline elapses.

See the rationale block in EmbeddedDb::open for the full context. The short version: SurrealKV holds a process-exclusive flock while a datastore is open, and Nauka’s CLI / forge daemon / announce listener all touch the same bootstrap.skv, so brief overlaps are normal. We retry the open rather than surface a hard error for every such overlap.
