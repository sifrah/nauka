ResourceDef lint engine.

Enforces structural, behavioral, and UX consistency across all resource definitions. Rules are checked at two points:

  1. lint_def() — called from .done(), validates a single ResourceDef. Errors panic immediately (binary won’t start with a broken def).
  2. lint_registry() — called from tests, validates cross-resource rules.

Severity levels:

  • Error — invariant violation, panics in .done().
  • Warning — convention violation, fails CI test.
