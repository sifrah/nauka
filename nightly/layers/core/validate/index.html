Input Validation

Nauka validates all user-supplied input at the boundary — in the CLI argument parser and in the API request handler — before any state changes are attempted. The validation rules are defined centrally so they are consistent everywhere. The CLI, the API, and any future interface all enforce the same rules.

Resource names

Section titled “Resource names”

Resource names must follow DNS label rules:

RuleConstraint
Length3 to 63 characters
CharactersLowercase letters (a-z), digits (0-9), and hyphens (-) only
StartMust start with a lowercase letter
EndMust end with a letter or digit
HyphensNo consecutive hyphens (--)

These rules match DNS label standards (RFC 1123), which matters because resource names may appear in DNS entries, WireGuard interface names, and Linux network interface names — all of which have similar constraints.

Regions and zones

Section titled “Regions and zones”

Regions and zones follow simpler rules:

RuleConstraint
Length1 to 32 characters
CharactersLowercase letters, digits, and hyphens only

Ports

Section titled “Ports”

Ports must be integers in the range 1 to 65535. Port 0 is not valid.

Port values can also be provided as strings (e.g., from environment variables), in which case they are parsed and validated in one step.

Durations

Section titled “Durations”

Duration values are parsed from human-readable strings with a numeric value followed by a unit:

UnitMeaningExample
sseconds60s
mminutes5m
hhours2h
ddays1d

Durations are used throughout the configuration for intervals and timeouts, such as health_check_interval = "60s" and join_timeout = "5m".

CIDR blocks

Section titled “CIDR blocks”

IPv4 CIDR blocks are validated for correct syntax and network alignment:

RuleConstraint
FormatA.B.C.D/N
OctetsEach must be 0-255
Prefix length0-32
Host bitsMust be zero (network address only)

When host bits are set, the error message suggests the correct network address, making it easy to fix the input.

Labels

Section titled “Labels”

Labels are key-value pairs in key=value format, used for tagging resources:

RuleConstraint
Key length1 to 63 characters
Key charactersLetters, digits, hyphens, dots, underscores
Key startMust start with a letter
Value length0 to 63 characters

Resource sizes

Section titled “Resource sizes”

Storage and compute sizes have platform-defined bounds:

ResourceMinimumMaximum
Disk size1 GB65,536 GB (64 TB)
Memory128 MB1,048,576 MB (1 TB)
vCPUs1256
