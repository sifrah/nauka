Every resource in Nauka has an **identity** that determines how it appears in the CLI and the API. The identity consists of three things: a canonical name, optional aliases, and a plural form.

## Resource name

The canonical name is the primary way you interact with a resource on the command line. It becomes the top-level subcommand under `nauka`:

```
nauka hypervisor list
nauka hypervisor get node-1
nauka hypervisor init --mesh-name production
```

In this example, `hypervisor` is the resource name. Names are always lowercase and may contain hyphens. Each registered resource must have a unique name.

A resource can also specify a separate **CLI name** if the command-line form should differ from the internal identifier. In most cases, the CLI name and the internal name are the same.

## Aliases

Aliases provide shorter alternatives for frequently used commands. A resource can have multiple aliases.

For example, if `hypervisor` has the alias `hv`, both of these commands are equivalent:

```
nauka hypervisor list
nauka hv list
```

Aliases are particularly useful for resources with long names that operators type often:

| Resource | Typical alias |
|----------|---------------|
| `hypervisor` | `hv` |
| `virtual-machine` | `vm` |
| `load-balancer` | `lb` |
| `firewall` | `fw` |

Aliases are shown in the CLI help output so operators can discover them.

## Plural form

The plural form is used in API route paths and in output messages like "No hypervisors found" or table headers.

By default, Nauka appends an `s` to the resource name. For resources where that produces an incorrect result, the plural is set explicitly:

| Resource | Default plural | Correct plural |
|----------|----------------|----------------|
| `hypervisor` | `hypervisors` | `hypervisors` (default works) |
| `policy` | `policys` | `policies` (must be set explicitly) |

The plural form controls the API path. Given a resource named `hypervisor` with plural `hypervisors`:

| CLI command | API equivalent |
|-------------|---------------|
| `nauka hypervisor list` | `GET /hypervisors` |
| `nauka hypervisor get node-1` | `GET /hypervisors/node-1` |
| `nauka hypervisor create` | `POST /hypervisors` |
| `nauka hypervisor delete node-1` | `DELETE /hypervisors/node-1` |

Changing the plural form affects the API paths but does not change the CLI command name. They are independent.

## Description

Each resource has a human-readable description that appears in CLI help output:

```
$ nauka --help
...
Commands:
  hypervisor  A bare-metal server in the mesh
  ...
```

This description is set once in the resource definition and used consistently across all generated help text.
