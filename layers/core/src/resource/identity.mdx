Every resource has an identity: a canonical name, optional aliases, and a plural form. These three things determine how the resource appears in the CLI and in the REST API.

## Name

The canonical name is how the resource is referred to everywhere. It becomes the top-level CLI subcommand and the singular form used in log messages and error text.

```rust
ResourceDef::build("hypervisor", "A bare-metal server in the mesh")
```

This produces the `nauka hypervisor` command family.

Names must be lowercase, use only letters and hyphens, and be unique across all registered resources. A resource named `load-balancer` becomes `nauka load-balancer ...` in the CLI.

## Aliases

Aliases let users type shorter versions of a command. A resource can have multiple aliases.

```rust
ResourceDef::build("hypervisor", "A bare-metal server in the mesh")
    .alias("hv")
```

With this alias, both `nauka hypervisor list` and `nauka hv list` work identically. Aliases also apply to API route matching, so `/hv` and `/hypervisors` both resolve to the same handler.

Aliases are useful for resources with long names that operators will type frequently. Common patterns:

| Resource | Alias |
|----------|-------|
| `hypervisor` | `hv` |
| `virtual-machine` | `vm` |
| `load-balancer` | `lb` |
| `firewall` | `fw` |

## Plural form

The plural form is used for API route paths and in table headers. By default, Nauka appends `s` to the name. For resources where that produces the wrong result, set it explicitly.

```rust
ResourceDef::build("gateway", "Network gateway")
    .plural("gateways")  // default: "gateways" — fine here

ResourceDef::build("policy", "IAM policy")
    .plural("policies")  // default would be "policys" — wrong
```

The plural is used in:

- REST API base path: `/gateways`, `/policies`
- Table header context lines: "Showing 4 gateways"
- Empty state messages: "No policies found"

## How identity maps to routes

Given a resource with name `vpc`, alias `v`, plural `vpcs`:

| CLI | API |
|-----|-----|
| `nauka vpc list` | `GET /vpcs` |
| `nauka v list` | `GET /vpcs` (alias resolves to same handler) |
| `nauka vpc get <id>` | `GET /vpcs/{id}` |
| `nauka vpc create` | `POST /vpcs` |
| `nauka vpc delete <id>` | `DELETE /vpcs/{id}` |

The plural form controls the API path. The name and aliases control the CLI. They are independent — changing the plural doesn't change the CLI command name.
