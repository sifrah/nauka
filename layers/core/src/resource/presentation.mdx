The presentation layer controls how resources appear in terminal output. When you run a `list` command, results are rendered as an aligned table. When you run a `get` command, the result is rendered as a key-value detail view. Both layouts are defined in the resource definition -- handlers just return data, and the framework handles formatting.

## Table output (list commands)

The table layout is defined as a list of columns. Each column maps a field from the response data to a display header:

```
$ nauka hypervisor list
NAME     ENDPOINT              STATUS     MESH
------------------------------------------------------
node-1   203.0.113.1:51820     active     production
node-2   203.0.113.2:51820     active     production
```

### Column configuration

Each column has several configurable properties:

**Width**: Columns can auto-size to their content, be fixed to a specific width, or be constrained to a minimum or maximum width.

**Alignment**: Columns are left-aligned by default. Numeric columns (like port numbers or sizes) can be right-aligned for easier scanning.

**Display format**: Values can be transformed before display (see Display Formats below).

### Empty state

When no resources are found, the table displays a configurable empty message instead of an empty table:

```
$ nauka hypervisor list
No hypervisors found. Run 'nauka hypervisor init' to create a mesh.
```

### Default sorting

Tables can specify a default sort field so results appear in a consistent, predictable order.

## Detail output (get commands)

The detail view is rendered as a two-column key-value layout, organized into sections:

```
$ nauka hypervisor get node-1

Mesh Information
================
  Name:            production
  Mesh ID:         mesh-01HW...
  Role:            founder

Network
================
  Endpoint:        203.0.113.1:51820
  IPv6 Address:    fd00:1234::1/128
  Listen Port:     51820
```

Detail views support multiple sections with optional titles. Each section contains a list of labeled fields. Fields without a value display a dash (`-`).

## Display formats

Display formats control how raw values are transformed before being shown to the user. The same formats work in both table columns and detail fields.

| Format | Input example | Output example |
|--------|--------------|----------------|
| **Plain** | `"production"` | `production` |
| **Bytes** | `1073741824` | `1.0 GiB` |
| **Duration** | `3661` | `1h 1m` |
| **Timestamp** | `1712345678` | `2024-04-05 18:34 UTC` |
| **Status** | `"active"` | `active` |
| **Masked** | `"wg-secret-key-abc123"` | `****...c123` |
| **YesNo** | `true` | `yes` |

### Bytes format

Automatically selects the appropriate unit (B, KiB, MiB, GiB) based on the value.

### Duration format

Converts seconds into a human-readable format using the largest appropriate units (days/hours, hours/minutes, minutes/seconds, or just seconds).

### Timestamp format

Converts Unix epoch timestamps into `YYYY-MM-DD HH:MM UTC` format.

### Masked format

Shows only the last 4 characters of sensitive values, replacing the rest with asterisks. Useful for keys and tokens.

### YesNo format

Converts boolean values into `yes` or `no` for clearer display.

## JSON output

All formatting is bypassed when `--json` is used. In JSON mode, the raw data from the handler is output directly as formatted JSON:

```
$ nauka hypervisor get node-1 --json
{
  "name": "node-1",
  "endpoint": "203.0.113.1:51820",
  "status": "active",
  ...
}
```

This makes it straightforward to use Nauka output in scripts and automation pipelines with tools like `jq`.
