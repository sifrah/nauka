The CLI formats raw numeric values into human-readable strings so operators can quickly understand when something happened, how long it took, and how much data is involved.

## Relative time

Timestamps in table output are displayed as relative durations, making it easy to see at a glance how recent an event is:

| Elapsed time | Displayed as |
|-------------|--------------|
| Less than 60 seconds | `just now` |
| 1 minute | `1 minute ago` |
| 2 -- 59 minutes | `N minutes ago` |
| 1 hour | `1 hour ago` |
| 2 -- 23 hours | `N hours ago` |
| 1 day | `1 day ago` |
| 2 -- 29 days | `N days ago` |
| 1 month | `1 month ago` |
| 2+ months | `N months ago` |

This format is used for fields like "last seen", "created", and "updated" in table and status output:

```
  NAME          STATUS       LAST SEEN
  ──────────────────────────────────────
  hv-west-01    ● running    2 hours ago
  hv-east-01    ● running    just now
  hv-south-01   ■ stopped    3 days ago
```

Timestamps in the future (due to clock skew) are displayed as `just now`.

## Duration formatting

Elapsed durations are formatted as compact human-readable strings:

| Seconds | Displayed as |
|---------|-------------|
| 0 | `0s` |
| 45 | `45s` |
| 60 | `1m` |
| 90 | `1m 30s` |
| 3600 | `1h` |
| 5400 | `1h 30m` |
| 86400 | `1d` |
| 90000 | `1d 1h` |

This format is used for uptime, operation duration, and timeout values:

```
  UPTIME     LAST RESTART
  1d 4h      3 days ago
```

Only the two most significant units are shown -- there is no `1d 2h 15m 30s` output. This keeps table columns narrow and output scannable.

## Byte formatting

Data sizes are formatted using binary (IEC) units:

| Bytes | Displayed as |
|-------|-------------|
| 0 | `0 B` |
| 512 | `512 B` |
| 1024 | `1.0 KiB` |
| 1,048,576 | `1.0 MiB` |
| 1,073,741,824 | `1.0 GiB` |

Values above 1024 bytes are shown with one decimal place of precision. This format is used for memory sizes, disk usage, and transfer amounts.
