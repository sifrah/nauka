When a `list` command returns results, the CLI renders them as an aligned terminal table. The table adapts to the terminal width, hides low-priority columns when space is tight, and applies status coloring automatically.

## What tables look like

```
  NAME          STATUS       MESH         ENDPOINT
  ───────────────────────────────────────────────────
  hv-west-01    ● running    production   10.0.1.1:51820
  hv-east-01    ● running    production   10.0.2.1:51820
  hv-south-01   ■ stopped    staging      10.0.3.1:51820
```

Headers are bold. Columns are separated by consistent spacing. There are no borders -- just clean, aligned columns with a thin separator line below the header.

## Automatic column sizing

Column widths are calculated based on the widest value in each column, including the header. The renderer measures all rows before printing, so columns are always wide enough for their content. When the terminal is wide enough, every column gets its natural width.

## Responsive column hiding

When the terminal is too narrow to fit all columns, the renderer removes columns starting with the lowest-priority ones. Each column has a priority level -- high-priority columns (like NAME and STATUS) are kept visible, while lower-priority columns (like ID or ENDPOINT) are hidden first.

This means operators always see the most important information, even on narrow terminals or when viewing output in a split pane.

## Truncation

When a column value is too long to fit its allocated width, it is truncated with an ellipsis:

```
  DESCRIPTION
  This is a long description that…
```

Truncation respects Unicode character widths, so multi-byte characters are handled correctly.

## Status coloring

Columns marked as status columns receive automatic color based on their value. See the [color documentation](/nauka/layers/core/ui/color) for the full mapping of status keywords to colors and indicators.

## Empty state

When a list returns no results, the table is not rendered -- no headers, no separator, no empty rows. The calling command is responsible for displaying an appropriate "no items found" message.

## Alternative output formats

The table renderer supports three output modes:

### Default (table)

The aligned, colored table format shown above. Used when stdout is a TTY.

### TSV (tab-separated values)

A machine-readable format with tab-separated columns and no headers, suitable for piping to `cut`, `awk`, or other text-processing tools:

```
hv-west-01	running	production	10.0.1.1:51820
hv-east-01	running	production	10.0.2.1:51820
```

ANSI color codes are stripped in TSV mode.

### JSON

A JSON array of objects where keys are derived from the lowercased header names (spaces replaced with underscores):

```json
[
  {
    "name": "hv-west-01",
    "status": "running",
    "mesh": "production",
    "endpoint": "10.0.1.1:51820"
  }
]
```

ANSI color codes are stripped in JSON mode.
