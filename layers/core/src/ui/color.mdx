The Nauka CLI uses a consistent color palette across all commands. Colors convey meaning -- green means healthy, red means broken -- so operators can scan output quickly without reading every word.

## Status indicators

Status values throughout the CLI are displayed with a colored dot prefix that communicates the resource state at a glance:

| Status | Indicator | Meaning |
|--------|-----------|---------|
| `running`, `available`, `active`, `healthy`, `attached`, `pass` | Green dot | Everything is operating normally |
| `creating`, `pending`, `scheduling`, `starting` | Blue circle | The resource is transitioning to a ready state |
| `stopped`, `decommissioned` | Dim square | The resource is intentionally inactive |
| `notready`, `degraded`, `draining` | Yellow triangle | Something needs attention |
| `failed`, `error`, `unreachable` | Red cross | The resource is in a broken state |

In a terminal with color support, the output looks like:

```
  NAME          STATUS
  ─────────────────────────
  hv-west-01    ● running
  hv-east-01    ◌ starting
  hv-south-01   ▲ degraded
  hv-north-01   ✖ unreachable
```

Status values that do not match any known keyword are displayed without a colored indicator.

## Text formatting

Beyond status indicators, the CLI uses a small set of text styles for consistent visual hierarchy:

- **Bold** -- used for table headers, resource names, and important labels
- **Green** -- success messages and confirmations
- **Red** -- error messages and failure indicators
- **Yellow** -- warnings and caution prompts
- **Blue** -- informational highlights and in-progress states
- **Dim** -- secondary information, optional fields, and de-emphasized labels

## NO_COLOR compliance

The CLI respects the `NO_COLOR` environment variable (see [no-color.org](https://no-color.org/)). When `NO_COLOR` is set to any value, all ANSI escape codes are suppressed and output is rendered as plain text:

```bash
NO_COLOR=1 nauka hypervisor status
```

Color is also automatically disabled when output is piped to a file or another command, preventing escape codes from appearing in logs or downstream tools.
