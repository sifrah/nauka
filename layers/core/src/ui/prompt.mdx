When a required argument is missing and the CLI is running in an interactive terminal, Nauka prompts the operator to supply the value rather than immediately returning an error. This makes the CLI comfortable to use without memorizing every flag, while remaining fully scriptable.

## How prompting works

After parsing command-line arguments, the CLI checks for missing required fields. If stdin is a TTY, it presents an interactive prompt for each missing value. If stdin is not a TTY (piped input, CI), the command exits with an error listing the missing arguments.

```
$ nauka hypervisor init

  Mesh name: production
  Region: eu-west
  Zone: fsn1

  ✓  Mesh initialized.
```

## Prompt types

The CLI chooses the prompt style based on the field's type.

### Text input

For string fields like names, CIDRs, IP addresses, paths, and durations:

```
  Mesh name: _
```

When a default value is available, it is shown and accepted by pressing Enter:

```
  Region [eu-west]: _
```

### Numeric input

For integer fields like port numbers, sizes, and counts. A default of `0` is shown when no other default is specified:

```
  Listen port [51820]: _
```

### Selection

For fields with a defined set of valid values (enums), the CLI presents an interactive selection list:

```
  Region:
  > eu-west
    us-east
    ap-south
```

The operator navigates with arrow keys and selects with Enter.

### Yes/No flag

For boolean fields:

```
  Enable monitoring? [y/N]: _
```

### Secret input

For sensitive values like API tokens or passwords, input is masked:

```
  API token: ********
```

The value is not echoed to the terminal and is not stored in shell history.

## Non-interactive contexts

In scripts, CI/CD pipelines, and any context where stdin is not a terminal, prompts are never shown. The CLI detects non-interactive mode automatically -- there is no `--no-prompt` flag.

In these contexts, always provide all required arguments explicitly:

```bash
nauka hypervisor init --name production --region eu-west --zone fsn1
```

If a required argument is missing in a non-interactive context, the CLI exits with an error indicating which argument is needed.

## Optional fields

Optional fields are never prompted. If an optional flag is not provided on the command line, it is left unset and the handler receives no value for that field. Prompting only occurs for required fields.
