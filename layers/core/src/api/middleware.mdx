Every request that reaches the Nauka API passes through a middleware stack before being dispatched to a resource handler. The middleware handles cross-cutting concerns that apply to all endpoints.

## Request ID

Every API response includes an `X-Request-Id` header with a unique identifier for that request:

```http
X-Request-Id: req-00000000002a
```

Request IDs are sequential counters formatted as 12-digit hexadecimal values, prefixed with `req-`. They increment for the lifetime of the server process and reset when the server restarts.

Include the request ID when reporting unexpected errors -- it allows the server operator to locate the exact request in the logs.

## Version header

Every response includes an `X-Nauka-Version` header containing the server's build version:

```http
X-Nauka-Version: 0.3.1
```

This lets clients detect version mismatches between the CLI and the API server without making a separate version-check call.

## Rate limiting

The API enforces a simple token-bucket rate limiter to prevent abuse. The default configuration allows 1000 requests per 60-second window.

When the limit is exceeded, the server returns `429 Too Many Requests`:

```json
{
  "code": "RATE_LIMITED",
  "message": "too many requests -- try again later"
}
```

The rate limiter resets its counter when the time window expires. The remaining request count is tracked internally but not currently exposed in response headers.

Rate limit parameters are configured at the server level:

| Parameter | Default |
|-----------|---------|
| Max requests per window | 1000 |
| Window duration | 60 seconds |

## Pagination

List endpoints accept pagination parameters via the query string:

```
GET /admin/v1/hypervisor?limit=25&offset=50&sort=name&order=desc
```

| Parameter | Default | Maximum | Description |
|-----------|---------|---------|-------------|
| `limit` | 50 | 200 | Number of items to return |
| `offset` | 0 | -- | Number of items to skip |
| `sort` | -- | -- | Field name to sort by |
| `order` | `asc` | -- | Sort direction: `asc` or `desc` |

If `limit` exceeds 200, it is silently clamped to 200.

Paginated responses include metadata to support cursor-based navigation:

```json
{
  "items": [ ... ],
  "count": 25,
  "total": 147,
  "offset": 50,
  "limit": 25,
  "has_more": true
}
```

- **count** -- the number of items in this page
- **total** -- the total number of items across all pages
- **has_more** -- whether additional items exist beyond this page

To fetch the next page, increment `offset` by the current `limit`:

```
GET /admin/v1/hypervisor?limit=25&offset=75
```
