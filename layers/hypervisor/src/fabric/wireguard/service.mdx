This is the canonical implementation of the WireGuard service management. It is responsible for installing `wireguard-tools`, generating the WireGuard configuration file, managing the `nauka-wg` systemd unit, and providing all service lifecycle operations. The `fabric::service` module re-exports this implementation for convenience.

## Files managed

| Path | Purpose | Permissions |
|---|---|---|
| `/etc/wireguard/nauka0.conf` | WireGuard interface configuration (`wg-quick` format) | `0600` (root only) |
| `/etc/systemd/system/nauka-wg.service` | systemd unit wrapping `wg-quick up/down` | Standard |

## Package installation

The `ensure_wireguard()` function checks for `wg-quick` on the system `PATH` and installs `wireguard-tools` automatically if missing. It detects the package manager in the following order:

1. `apt-get` (Debian / Ubuntu) -- runs `apt-get update -qq` first
2. `dnf` (Fedora / RHEL 8+)
3. `yum` (CentOS / older RHEL)
4. `pacman` (Arch Linux)
5. `apk` (Alpine Linux)
6. `zypper` (openSUSE)

If no supported package manager is detected, the function returns an error instructing the operator to install `wireguard-tools` manually.

After installation, the function verifies that `wg-quick` is now available. If it is still not found (e.g., the package installed but the binary is in an unexpected location), the operation fails with a clear error.

## Configuration generation

The WireGuard configuration is generated from the node's identity and peer list:

```ini
[Interface]
PrivateKey = <this node's private key>
ListenPort = 51820
Address = fd12:3456:789a::1/128

[Peer]
PublicKey = <peer's public key>
AllowedIPs = fd12:3456:789a::2/128
PersistentKeepalive = 25
Endpoint = 203.0.113.20:51820
```

Each peer gets a `[Peer]` section with:
- `AllowedIPs` set to the peer's single mesh IPv6 (`/128`) -- only mesh traffic goes through the tunnel
- `PersistentKeepalive = 25` -- ensures regular handshakes for health monitoring
- `Endpoint` included only when known

## The systemd unit

The unit file uses `Type=oneshot` with `RemainAfterExit=yes`, which means `wg-quick up` runs once to create the interface, and systemd considers the service "active" as long as the interface exists:

- `ExecStart` runs `wg-quick up /etc/wireguard/nauka0.conf`
- `ExecStop` runs `wg-quick down /etc/wireguard/nauka0.conf`
- The service starts after `network-online.target` to ensure network availability
- `WantedBy=multi-user.target` enables automatic start on boot

## Lifecycle operations

| Function | What it does |
|---|---|
| `install()` | Creates `/etc/wireguard/` directory, writes `nauka0.conf` (with `0600` permissions), writes the systemd unit, runs `systemctl daemon-reload` |
| `enable_and_start()` | Runs `systemctl enable --now nauka-wg` |
| `start()` | Runs `systemctl start nauka-wg` |
| `stop()` | Runs `systemctl stop nauka-wg` |
| `uninstall()` | Stops and disables the service, removes both files, reloads the systemd daemon |
| `update_config()` | Rewrites `nauka0.conf` and applies changes via `wg syncconf nauka0 /etc/wireguard/nauka0.conf`. Falls back to a full service restart if `syncconf` fails. |

## Live configuration updates

The `update_config()` function is called whenever the peer list changes (new join, leave, or announcement). It rewrites the configuration file and uses `wg syncconf` to apply changes without bringing the interface down. This is non-disruptive -- existing tunnels continue running.

If `wg syncconf` fails for any reason, the function falls back to restarting the entire service via `systemctl restart nauka-wg`. This ensures the WireGuard interface always reflects the current state, even in edge cases.

## Status checks

| Check | Method |
|---|---|
| `is_installed()` | Checks if `/etc/systemd/system/nauka-wg.service` exists on disk |
| `is_active()` | Runs `systemctl is-active --quiet nauka-wg` |
| `is_enabled()` | Runs `systemctl is-enabled --quiet nauka-wg` |
| `wg_quick_available()` | Checks if `wg-quick` is on the system `PATH` |

All status checks are non-destructive and return booleans without throwing errors.
