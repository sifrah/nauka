The Nauka control plane is a distributed, replicated key-value store built on [TiKV](https://tikv.org/). It stores cluster-wide state that must survive individual node failures and be accessible from any node in the mesh. All control plane traffic flows over the encrypted WireGuard mesh -- no control plane ports are exposed to the public internet.

## Components

The control plane consists of two services:

**PD (Placement Driver)** manages cluster membership, shard routing, and leader election using the Raft consensus algorithm. PD exposes an HTTP API on port 2379 for health checks, membership management, and store discovery.

**TiKV** is the storage engine. It persists data in RocksDB, replicates it across nodes using Raft, and registers itself with PD on startup. TiKV listens on port 20160.

## Cluster topology

Nauka manages the cluster topology automatically based on join order. PD uses Raft consensus, which works best with an odd number of members. Nauka caps PD membership at **3 nodes**.

| Node | Runs | Rationale |
|---|---|---|
| 1st (init) | PD + TiKV | Bootstrap node starts the Raft group |
| 2nd joiner | PD + TiKV | Grows the Raft group to 2 members |
| 3rd joiner | PD + TiKV | Raft group reaches 3 (optimal quorum) |
| 4th+ joiners | TiKV only | PD cluster is full; additional nodes contribute storage |

This means even a single-node cluster runs PD alongside TiKV. There is no migration needed when scaling up -- PD is ready from day one.

<Aside type="note">
The decision about whether a joining node runs PD is made locally based on the number of peers known at join time, without querying the PD API. This avoids race conditions during rapid parallel joins.
</Aside>

## Network ports

All ports bind exclusively to the node's mesh IPv6 address. They are not reachable from outside the WireGuard mesh.

| Port | Service | Protocol | Purpose |
|---|---|---|---|
| 2379 | PD | HTTP | Client API (TiKV registration, health checks, store discovery) |
| 2380 | PD | HTTP | Peer communication (Raft consensus between PD members) |
| 20160 | TiKV | gRPC | Key-value storage operations |

## Installation

TiKV and PD are installed automatically via [TiUP](https://tiup.io/) during `nauka hv init` or `nauka hv join`. No manual installation is required. TiUP is placed at `/opt/nauka/tiup/` and manages PD and TiKV as its components.

## File paths

| Path | Purpose |
|---|---|
| `/opt/nauka/tiup/` | TiUP home directory (binaries, components) |
| `/etc/nauka/pd.toml` | PD configuration |
| `/etc/nauka/tikv.toml` | TiKV configuration |
| `/var/lib/nauka/pd/` | PD data directory (Raft logs, metadata) |
| `/var/lib/nauka/tikv/` | TiKV data directory (RocksDB) |
| `/var/log/nauka/pd.log` | PD log file (50 MB rotation) |
| `/var/log/nauka/tikv.log` | TiKV log file (50 MB rotation) |
| `/etc/systemd/system/nauka-pd.service` | PD systemd unit |
| `/etc/systemd/system/nauka-tikv.service` | TiKV systemd unit |
