The `doctor` command inspects all four sub-systems on the current node and prints a color-coded report. It is the single fastest way to verify that a node is healthy or to narrow down which layer has a problem.

```sh
nauka hv doctor
```

## How it works

Doctor loads the node's local state, then runs checks sequentially across four sections: Fabric, Controlplane, Storage, and System. Each individual check reports one of four statuses:

| Status | Symbol | Meaning |
|---|---|---|
| OK | green check | Check passed |
| WARN | yellow exclamation | Degraded but not broken |
| FAIL | red cross | Something is wrong and needs attention |
| SKIP | grey dash | Sub-system not installed or not configured |

Skipped checks do not count as failures. A node that has not installed the control plane will show SKIP for all controlplane checks rather than FAIL.

## Fabric checks

These checks verify the WireGuard mesh networking layer.

| Check | Pass condition | Fail / Warn condition |
|---|---|---|
| wireguard-tools | `wg` command is available on PATH | Not installed (FAIL) |
| wg config | `/etc/wireguard/nauka0.conf` exists | Missing (FAIL) |
| fabric network | Network interface is up | Interface down (FAIL) |
| fabric service | systemd service is active | Service stopped (WARN) |
| peers | All configured peers are reachable | Some peers unreachable (WARN) |
| mesh ping | ICMPv6 ping to up to 3 peers responds | Some pings fail (WARN) |

In `direct` mode, the wireguard-tools and wg config checks are replaced by a simple "direct (no tunnel)" confirmation. If no peers are configured (freshly initialized node), the peer checks report OK.

## Controlplane checks

These checks verify the TiKV distributed key-value store and its Placement Driver.

| Check | Pass condition | Notes |
|---|---|---|
| tiup | `/opt/nauka/tiup/bin/tiup` exists | If missing, all controlplane checks are skipped |
| pd service | `nauka-pd` systemd service is active | Shows "not running (tikv-only node)" as OK if the unit is not installed (4th+ node) |
| tikv service | `nauka-tikv` systemd service is active | FAIL if the unit exists but is stopped |
| pd health | All PD members report healthy via the PD API (`/pd/api/v1/health`) | WARN if some members are unhealthy |
| pd leader | A PD leader has been elected | FAIL if no leader |
| tikv stores | PD API reports registered TiKV stores (`/pd/api/v1/stores`) | Shows count of registered stores |

The PD health and leader checks query the PD HTTP API at `http://[mesh_ipv6]:2379`. If the API is unreachable, those checks report WARN rather than FAIL.

## Storage checks

These checks verify the ZeroFS S3-backed storage layer.

| Check | Pass condition | Notes |
|---|---|---|
| zerofs | `/usr/local/bin/zerofs` binary exists | SKIP if not installed |
| region `<name>` | `nauka-zerofs-<name>` service is active | WARN if configured but stopped; shows S3 bucket name |

If no regions are configured, doctor reports "none configured" as OK.

## System checks

These checks verify host-level resources.

| Check | Pass condition | Warn / Fail threshold |
|---|---|---|
| disk space | `/var/lib/nauka` has free space | WARN below 1 GB, FAIL below 256 MB |
| logs | `/var/log/nauka` exists and is writable | WARN if missing or not writable |
| state file | `~/.nauka/hypervisor.json` exists and is valid JSON | FAIL if corrupt, SKIP if not initialized |

## Reading the output

After all sections, doctor prints a summary line:

```
  3 passed, 1 warnings, 0 errors
```

<Aside type="tip">
Use `nauka hv doctor` as a post-deploy smoke test in your runbooks. Run it after `nauka hv init`, after `nauka hv join`, and during incident triage to quickly isolate which layer is affected.
</Aside>

## When to run doctor

- **After initializing a node** -- confirm all three sub-systems came up correctly
- **After joining a cluster** -- verify mesh connectivity, control plane membership, and storage setup
- **During incidents** -- determine which sub-system is degraded before diving into logs
- **Periodically** -- as a health check in monitoring scripts or cron jobs
