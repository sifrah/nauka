Configuration

Nauka’s configuration system has a clear priority chain: CLI flags beat environment variables, which beat the config file, which beats built-in defaults. You never need to worry about which source “wins” — higher-priority sources always override lower ones.

Config file location

Section titled “Config file location”

The config file lives at ~/.nauka/config.toml. This is created automatically by nauka init with sensible defaults.

Environment variables

Section titled “Environment variables”

Every config key has a corresponding environment variable with the NAUKA_ prefix. Nested keys use _ as the separator.

Config keyEnvironment variable
daemon.log_levelNAUKA_DAEMON_LOG_LEVEL
wireguard.listen_portNAUKA_WIREGUARD_LISTEN_PORT
peering.timeoutNAUKA_PEERING_TIMEOUT
storage.data_dirNAUKA_STORAGE_DATA_DIR
logging.formatNAUKA_LOGGING_FORMAT

This makes Nauka easy to configure in containerized or CI environments without writing config files.

Priority chain

Section titled “Priority chain”

A value is only taken from a source if it was explicitly set. An absent CLI flag does not override a config file value with an empty string.

Configuration sections

Section titled “Configuration sections”

daemon

Section titled “daemon”

Controls the Nauka process itself.

KeyDefaultDescription
socket_path~/.nauka/nauka.sockUnix socket path for CLI-daemon communication
log_levelinfoLogging verbosity: trace, debug, info, warn, error

wireguard

Section titled “wireguard”

Controls WireGuard interface setup.

KeyDefaultDescription
listen_port51820UDP port WireGuard listens on
dns["1.1.1.1"]DNS resolvers for mesh peers

peering

Section titled “peering”

Controls mesh peer discovery and join.

KeyDefaultDescription
timeout30sTime to wait for a peer to respond during join
retry_interval5sInterval between retry attempts

storage

Section titled “storage”

Controls local state storage.

KeyDefaultDescription
data_dir~/.nauka/dataDirectory for the embedded database and working files
cache_size_mb512In-memory cache for the embedded database

logging

Section titled “logging”

Controls log output.

KeyDefaultDescription
formattexttext for human-readable, json for structured
filenonePath to write logs to (in addition to stderr)
rotationdailyLog rotation frequency: hourly, daily, never

Validation on load

Section titled “Validation on load”

All config values are validated when the config is loaded. Invalid values cause the process to exit with a clear error message before doing any work.

Duration values use the same human-readable format as the CLI — 30s, 5m, 2h. The config file does not use raw seconds or milliseconds.
